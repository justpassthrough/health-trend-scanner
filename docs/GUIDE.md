## 프로젝트 개요

약사 블로그용 글감 자동 발굴 대시보드.
매일 4회 자동으로 건강/의약 분야 트렌드를 스캔하여 점수를 매기고, 모바일 대시보드로 보여줌.

- **GitHub 레포**: https://github.com/justpassthrough/health-trend-scanner
- **대시보드 URL**: https://justpassthrough.github.io/health-trend-scanner/
- **자동 갱신**: 매일 07:30 / 13:00 / 19:00 / 00:00 (한국시간)

## 핵심 공식

```
글감 점수(Topic Score) = H × I × P × C × Y
```

곱셈 구조이므로 어느 하나가 0이면 전체 점수가 0.

| 변수 | 이름 | 범위 | 의미 |
|------|------|------|------|
| **H** | 주제 온도 | 0 ~ 100 | 검색량이 얼마나 급등했는지. 0이면 나머지 계산 안 함 |
| **I** | 검색의도 배수 | 0.5 ~ 1.5 | 의심/경고형이면 유리, 쇼핑/추천형이면 불리 |
| **P** | 약사근거 배수 | 0.7 ~ 1.3 | 약사 전문성이 필요한 주제인지 |
| **C** | 경쟁도 배수 | 0.5 ~ 1.2 | 블로그 경쟁이 적을수록 유리 |
| **Y** | 유튜브 선행 배수 | 1.0 ~ 1.3 | 유튜브에서 먼저 터졌으면 보너스 |

### 판정 기준

| 점수 | 판정 | 대시보드 표시 |
|------|------|-------------|
| 80 이상 | 지금 써라 | 초록 뱃지 |
| 40 ~ 79 | 쓸 만하다 | 노랑 뱃지 |
| 15 ~ 39 | 앵글 바꾸면 가능 | 회색 뱃지 |
| 15 미만 | 패스 | 어두운 뱃지 |

## 각 변수 산출 방법 (collect.py)

### H — 주제 온도

**데이터 소스**: 네이버 DataLab API (일간 검색량)

**산출 방식**:
1. 최근 28일 일간 데이터 조회
2. 최근 7일 평균 vs 이전 7일 평균 비교
3. 변화율 = (최근7일 - 이전7일) / 이전7일 × 100

**변화율 → H 점수 변환표**:

| 변화율 | H |
|--------|---|
| +300% 이상 | 100 |
| +200% 이상 | 70 |
| +100% 이상 | 50 |
| +50% 이상 | 30 |
| +20% 이상 | 15 |
| 0% 초과 | 5 |
| 0% 이하 | 0 |

**보조 로직**: DataLab 변화율이 0이어도 뉴스에 자주 등장하면 최소 점수 부여
- 뉴스 5건 이상 → H = 최소 15
- 뉴스 3건 이상 → H = 최소 10

**코드 위치**: `collect.py` → `get_search_trend()`, `calc_h_score()`

**수정하고 싶을 때**:
- 변화율 기준을 바꾸려면 → `calc_h_score()` 함수의 if/elif 조건문 수정
- 비교 기간을 바꾸려면 → `get_search_trend()` 함수의 `timedelta(days=28)`, `points[-7:]`, `points[-14:-7]` 수정

---

### I — 검색의도 배수

**데이터 소스**: 네이버 블로그 검색 API → 동반 텍스트 분석

**공식**:
```
I = 1.0 + 0.5 × (A - B) / (A + B + 1)

A = 의심/경고형 단어 매칭 수
B = 쇼핑/추천형 단어 매칭 수
```

**의심/경고형 단어 (A)**:
```python
["부작용", "위험", "진짜", "효과없", "먹어도되나", "품절", "대체",
 "주의", "경고", "논란", "가짜", "허위", "과대광고", "리콜", "중단"]
```

**쇼핑/추천형 단어 (B)**:
```python
["추천", "가격", "할인", "후기", "구매", "최저가", "쿠팡",
 "순위", "비교", "TOP", "best", "입문"]
```

- A > B → 1.5에 수렴 (약사 블로그에 유리)
- A < B → 0.5에 수렴 (약사 블로그에 불리)
- A ≈ B → 1.0

**코드 위치**: `collect.py` → `calc_i_score()`

**수정하고 싶을 때**: `DOUBT_KEYWORDS`, `SHOPPING_KEYWORDS` 리스트에 단어 추가/제거

---

### P — 약사근거 배수

**데이터 소스**: 네이버 블로그 동반 텍스트 (I와 같은 데이터 재사용)

**공식**:
```
P = 0.7 + 0.6 × min(n, 3) / 3

n = 약사 전문성 키워드 매칭 수
```

**약사 전문성 키워드**:
```python
["처방", "복용법", "용량", "상호작용", "금기", "성분", "함량",
 "부작용", "약동학", "반감기", "대사", "CYP", "약물"]
```

- 3개 이상 매칭 → P = 1.3 (최대)
- 0개 매칭 → P = 0.7 (최소)

**코드 위치**: `collect.py` → `calc_p_score()`

**수정하고 싶을 때**: `PHARMA_KEYWORDS` 리스트에 단어 추가/제거

---

### C — 경쟁도 배수

**데이터 소스**: 네이버 블로그 검색 API

**핵심**: 키워드 단독이 아니라 **"키워드 + 세부키워드" 조합**으로 검색

**산출 과정**:
1. 블로그 동반 텍스트에서 세부키워드 상위 2개 추출 (예: "백신" → "접종", "안전성")
2. "백신 접종", "백신 안전성" 각각으로 블로그 검색
3. 최근 7일 내 게시글 수가 가장 적은 조합을 대표값으로 사용
4. 이 조합이 **가장 블루오션인 앵글**

**게시글 수 → C 배수 변환**:

| 7일 내 게시글 수 | C | 라벨 |
|-----------------|---|------|
| 0 ~ 5건 | 1.2 | 블루오션 |
| 6 ~ 20건 | 1.0 | 보통 |
| 21 ~ 50건 | 0.7 | 경쟁 많음 |
| 50건 초과 | 0.5 | 레드오션 |

**코드 위치**: `collect.py` → `extract_sub_keywords()`, `_count_recent_blogs()`, `calc_c_score()`

**수정하고 싶을 때**:
- 게시글 수 기준을 바꾸려면 → `calc_c_score()` 함수의 if/elif 조건문 수정
- 세부키워드 추출 로직 → `extract_sub_keywords()` 함수

---

### Y — 유튜브 선행 배수

**데이터 소스**: YouTube Data API v3

**산출 방식**:
1. 해당 키워드로 최근 48시간 내 영상 검색
2. 조회수 확인

| 조건 | Y |
|------|---|
| 10만+ 조회 영상 존재 | 1.3 |
| 3만~10만 조회 영상 존재 | 1.15 |
| 없음 | 1.0 |

**코드 위치**: `collect.py` → `search_youtube()`

**참고**: 실시간 갱신(수동 버튼) 시에는 유튜브 API 호출을 스킵하고 이전 데이터를 재사용 (`--skip-youtube` 플래그)

## 데이터 수집 파이프라인 (collect.py)

실행 순서:

```
1. 네이버 뉴스 수집 → 급등 키워드 자동 추출 (상위 30개)
   ↓
2. 각 키워드별:
   ├── DataLab API → 변화율 → H 점수
   ├── (H=0이면 스킵)
   ├── 네이버 뉴스 검색 → 관련 헤드라인 2개 (왜 급등했는지)
   ├── 블로그 검색 → 동반 텍스트 분석 → I, P 점수
   ├── 세부키워드 추출 → "키워드+세부키워드" 블로그 검색 → C 점수
   └── YouTube 검색 → Y 점수
   ↓
3. 최종 점수 = H × I × P × C × Y
   ↓
4. 약업계 뉴스 수집 (식약처, 허가, 리콜, 공급중단 등)
   ↓
5. 유튜브 급등 영상 탐지 (네이버에 아직 안 온 것)
   ↓
6. data/latest.json 저장
7. data/history/{timestamp}.json 히스토리 누적 저장
```

### 키워드 자동 추출 방식

사전 등록 키워드가 **아님**. 매 실행마다 자동 발굴:

1. 15개 시드 쿼리로 네이버 뉴스 검색 (건강기능식품, 영양제, 비타민 등)
2. 수집된 뉴스 제목에서 한글 명사 추출
3. 불용어 제거 후 빈도 3회 이상인 단어 = 트렌딩 키워드
4. 상위 30개만 분석 (API 호출량 절약)

**시드 쿼리 수정**: `collect.py` → `extract_keywords_from_news()` 함수의 `seed_queries` 리스트

## 대시보드 (build_dashboard.py)

### 3개 영역

| 영역 | 역할 | 시간축 |
|------|------|--------|
| 🔥 오늘의 글감 | 지금 터지고 있는 것 + 점수 판정 | 현재 |
| 🎬 유튜브 급등 | 유튜브에서 터졌지만 네이버에 아직 안 온 것 | 2~5일 후 예측 |
| 📰 약업계 헤드라인 | 아직 일반인 검색에 안 잡히는 업계 내부 이슈 | 선행 지표 |

### 글감 카드에 표시되는 정보

```
#1 캡박시브                          60.7 🟡 쓸만하다
   ↑ 상승  45.2 → 60.7                          ← 추이 (히스토리 기반)
   📰 MSD, 성인 전용 21가 폐렴구균 백신 출시      ← 왜 급등했는지
   급등 +108%  의심/경고형  "캡박시브 백신" 3건 (블루오션)  ← 지표들
   처방 복용법                                   ← 약사 전문성 키워드
```

### 추이 라벨

히스토리(최근 48시간)를 기반으로 점수 변화 표시:

| 라벨 | 조건 | 색상 | 판단 |
|------|------|------|------|
| ↑↑ 상승 중 | 직전 2회 대비 연속 상승 | 초록 | 지금 쓰면 타이밍 좋음 |
| ↑ 상승 | 직전 대비 3점 초과 상승 | 초록 | 유망 |
| → 유지 | 변동 3점 이내 | 회색 | 피크일 수 있음, 확인 필요 |
| ↓ 하락 | 직전 대비 3점 초과 하락 | 빨강 | 늦었을 가능성 |
| ↓↓ 하락 중 | 직전 2회 대비 연속 하락 | 빨강 | 패스 권장 |
| 🆕 신규 | 이전 히스토리 없음 | 파랑 | 방금 떠오른 이슈, 선점 기회 |

**코드 위치**: `build_dashboard.py` → `calc_trend()`

**수정하고 싶을 때**: `calc_trend()` 함수의 `diff > 3`, `diff < -3` 임계값 수정

### 실시간 갱신 버튼

대시보드 상단에 "실시간 갱신" 버튼이 있음.

## 히스토리 시스템

### 저장 방식

매 실행마다 `data/history/{YYYY-MM-DD_HHMM}.json` 파일 생성.
내용은 `data/latest.json`과 동일한 구조.

### 용량 추정

- 1회 실행 결과: ~50KB
- 하루 4회 × 365일 = 약 73MB/년
- GitHub 레포 용량 한도 1GB 대비 충분

### 추이 분석

`build_dashboard.py`의 `load_history(hours=48)` 함수가 최근 48시간 히스토리를 로드하여
각 키워드별 점수 이력을 추적.

**추이 분석 기간 변경**: `load_history(hours=48)` → 원하는 시간으로 변경
