## 1. 프로젝트 개요

**목적**: 병원 약사이자 DDS 연구자가 운영하는 네이버 건강 블로그의 **새로운 글감을 AI로 발굴**하는 도구.

**핵심 변화 (v1 → v2)**:
- v1은 뉴스 제목에서 명사를 추출하고 빈도를 세서 점수를 매기는 방식 → "비만", "중단" 같은 의미 없는 단어 파편만 나옴
- v2는 **AI(Claude Haiku)가 뉴스 기사를 직접 읽고, 블로거의 기존 글과 대조하여, 구체적인 글감을 맥락과 함께 추천**

**사용자**: 네이버 블로그 `biopharmblog` 운영자 (병원 약사 + DDS 연구자)

## 4. 핵심 알고리즘: AI 프롬프트

### 4-1. 프롬프트 구조 (build_ai_prompt 함수, collect_v2.py:272)

AI에게 전달하는 데이터:
1. **역할 설정**: "병원 약사이자 DDS 연구자의 콘텐츠 전략가"
2. **내 블로그 글 제목 목록**: my_posts.json에서 최근 50개
3. **오늘의 뉴스**: 4개 카테고리별로 각 최대 25개 (제목 + 설명문 100자)
4. **출력 규칙**: JSON 배열, 8~15개 항목

### 4-2. AI에게 주는 우선순위 규칙

```
1. 내 블로그에 전혀 없는 영양제/성분 이름 → 최우선
2. 소비자에게 직접 영향 주는 약업계 뉴스 (건보, 품절, 리콜) → 높음
3. 갑자기 검색량 급등하는 성분/건강 이슈 → 높음
4. 이미 다룬 주제이지만 새 이슈가 터진 경우 → 보통
5. "비만", "건강" 같은 포괄적 키워드는 제외
```

### 4-3. AI 출력 형식 (각 항목)

```json
{
  "keyword": "구체적 키워드",
  "category": "영양제·성분 | 약업계·정책 | 질환·치료 | 소비자건강",
  "why_now": "왜 지금 써야 하는지 2~3문장",
  "pharmacist_angle": "약사/DDS 연구자 차별화 앵글",
  "title_idea": "블로그 글 제목 아이디어",
  "already_covered": false,
  "covered_posts": [],
  "source_headlines": ["근거 뉴스 제목"]
}
```

### 4-4. JSON 파싱 (run_ai_analysis 함수, collect_v2.py:343)

AI 응답에서 JSON을 추출하는 3단계:
1. `` ```json ... ``` `` 감싸기 제거
2. `[` 로 시작하지 않으면 배열 부분만 정규식 추출
3. `json.loads()` 파싱

## 6. 내 블로그 글 목록 연동

**데이터 소스**: `inflow-keyword-analyzer` 레포의 `data/my_posts.json`
- URL: `https://raw.githubusercontent.com/justpassthrough/inflow-keyword-analyzer/main/data/my_posts.json`
- 형식: `{ "blog_id": "biopharmblog", "posts": [ { "title": "...", "url": "...", "date": "...", "body_keywords": [...] } ] }`
- 갱신: 매일 KST 08:00 GitHub Actions에서 RSS + 스크래핑으로 자동 갱신

**사용 방식**:
- AI 프롬프트에 글 제목 최근 50개를 전달
- AI가 "이 뉴스 이슈는 기존 글 X와 관련 있다" → `already_covered=true`, `covered_posts=["X"]`
- 사용자가 별도로 관리할 필요 없음

## 7. 보강 데이터

AI가 뽑은 15개 후보에 대해서만 추가 데이터를 수집합니다.

### 7-0. 핵심 키워드 추출 (_extract_core_keyword, collect_v2.py:438)

AI가 뽑는 키워드가 "벤포티아민 (활성비타민 B1)"처럼 괄호 설명이 붙는 경우가 많아 API 검색 적중률이 낮습니다.
이를 해결하기 위해 API 조회 전에 핵심 키워드만 추출합니다.

- 괄호 내용 제거: `"벤포티아민 (활성비타민 B1)"` → `"벤포티아민"`
- 구분자 분리: `"셀레늄 + 아연"` → `"셀레늄"` (첫 번째 요소)
- 핵심 키워드로 조회 후 결과가 0이면 원래 키워드로 재시도

### 7-0.5. 뉴스 건수 + 헤드라인 (get_news_count_and_headlines, collect_v2.py:443)

- **API**: Naver News Search (`/v1/search/news.json`)
- **뉴스 건수**: API 응답의 `total` 필드 사용 (실제 총 건수, 상한 없음)
- **헤드라인**: 상위 3개 뉴스 제목 + 링크 반환 (대시보드에서 클릭 가능)
- `display=3`으로 요청하여 데이터 전송량 최소화

### 7-1. 검색량 트렌드 (get_search_trend, collect_v2.py:169)

- **API**: Naver DataLab (`/v1/datalab/search`)
- **기간**: 최근 28일
- **계산**: 최근 7일 평균 vs 이전 7일 평균 변화율
- **판정**: 급상승(>50%) / 상승(>10%) / 유지(-10%~10%) / 하락(<-10%)

### 7-2. 전문가 갭 (get_expert_gap, collect_v2.py:219)

- **API**: Naver Blog Search (`/v1/search/blog.json`)
- **계산**: 전체 블로그 수 / ("키워드 + 약사" 블로그 수 + 1)
- **판정**:
  - total < 100 → "수요 적음"
  - gap_ratio ≥ 30 → "전문가 갭 큼" (좋은 기회)
  - gap_ratio ≥ 10 → "전문가 부족"
  - gap_ratio ≥ 3 → "보통"
  - gap_ratio < 3 → "전문가 포화"

### 7-3. 연속 등장일수 (load_scan_history, collect_v2.py:464)

- 최근 7일간 스캔 히스토리에서 같은 키워드가 며칠 연속 등장했는지 계산
- 2일 이상이면 대시보드에 "N일 연속" 배지 표시
- 일시적 이슈 vs 지속적 관심사 구분용

## 9. 대시보드 (build_dashboard_v2.py)

### 9-1. 작동 방식

- `data/scans/*.json` 최근 28개를 모두 읽어서 HTML 내 JS 변수로 임베드
- 클라이언트 사이드에서 날짜 네비게이션 처리 (추가 API 호출 없음)
- 정적 HTML 1개 파일 (GitHub Pages에서 바로 배포)

### 9-2. UI 구성

```
┌─ 헤더: 건강 트렌드 스캐너 v2 ─────────────────────┐
├─ 날짜 네비게이터: [< 이전] [3/12 15:47 ▼] [다음 >] ┤
├─ 통계 바: 🆕 새 글감 11개 | 🔥 기존주제 4개 | 📰 839건 ┤
├─ 카테고리 필터: [전체] [영양제·성분] [약업계·정책] ... ┤
├─ 🆕 새 글감 발굴 섹션 ─────────────────────────────┤
│  카드: 키워드 + 카테고리 태그                         │
│        [이슈 메트릭 바] 뉴스 N건 | 검색량 X | 변화 +Y% | 전문가갭 │
│        💡 왜 지금? (2~3줄)                          │

│  카드: [이미 작성 N편] 배지 + 관련 기존 글 목록        │
├─ 푸터: 비용 + 날짜시간 ────────────────────────────┤
```

### 9-3. 카테고리별 색상

| 카테고리 | 색상 | CSS 클래스 |
|---|---|---|
| 영양제·성분 | 초록 (#3fb950) | `.cat-supplement` |

### 9-4. 이슈 메트릭 바 (buzz-bar)

각 카드 상단에 색상코딩된 메트릭 바가 표시됩니다:

| 지표 | 빨강 (🔥 hot) | 노랑 (warm) | 회색 (cool) |
|---|---|---|---|
| 뉴스 건수 | ≥ 50건 | ≥ 10건 | < 10건 |
| 검색량 (주간평균) | ≥ 50 | ≥ 20 | < 20 |
| 변화율 | ≥ +50% | ≥ +10% | < +10% |
| 전문가갭 | 갭 큼 | 전문가 부족 | 보통/포화 |

### 9-5. 특수 배지

- **N일 연속**: 연속 2일 이상 등장 시 파란색 배지
- **이미 작성 N편**: already_covered=true일 때 노란색 배지

## 14. 알려진 한계 & 향후 개선 가능성

### 현재 한계

1. **검색 트렌드 0%**: AI가 뽑는 키워드가 구체적(예: "벤포티아민")이라 DataLab에서 데이터가 없는 경우 많음
   - 대응 완료: `_extract_core_keyword()`로 괄호/구분자 제거 후 조회, 0이면 원래 키워드로 재시도
   - 추가 개선안: 키워드를 상위 카테고리("비타민B")로 변환하여 트렌드 조회

2. **비용 예상 초과**: 설계 시 $0.015/회 예상 → 실제 $0.05/회
   - 원인: AI 출력이 예상보다 상세함 (6,769 토큰 vs 예상 2,250)
   - 개선안: 프롬프트에서 why_now/pharmacist_angle을 "각 1문장"으로 제한

3. **AI 출력 불안정**: 첫 실행 시 JSON 파싱 실패 발생
   - 현재 대응: 3단계 파싱 (코드블록 제거 → 배열 추출 → json.loads)
   - 추가 대응 가능: 재시도 로직 또는 structured output 사용

4. **전문가갭 정확도**: "키워드 약사" 블로그 검색이 부정확할 수 있음
   - 예: "담석증 약사" 검색 시 담석증 글이 아닌 약사 관련 다른 글이 잡힘

5. **씨드 쿼리 고정**: 현재 37개 고정 쿼리 → 새로운 카테고리 트렌드를 놓칠 수 있음
   - 개선안: AI가 이전 스캔에서 발견한 트렌드를 기반으로 씨드 쿼리를 동적으로 조정

### 가능한 개선 방향

- **키워드 구체화**: AI에게 "블로그 제목에 바로 쓸 수 있는 수준"으로 키워드를 더 구체적으로 요청
- **이미 작성 판단 강화**: 현재 AI의 판단에 의존 → body_keywords까지 프롬프트에 포함하면 정확도 향상
- **검색량 조회 개선**: 긴 키워드 → 핵심 단어 추출 후 DataLab 조회
- **스캔 빈도 조정**: 하루 2회가 과한지 / 부족한지 사용 후 판단
- **대시보드 개선**: 카드 접기/펼치기, 즐겨찾기, "글 작성 완료" 체크 기능

## 16. 디버깅 가이드

### AI 분석 실패 시

1. GitHub Actions 로그에서 "응답 첫 200자" 확인
2. JSON 파싱 에러가 대부분 → `run_ai_analysis()`의 정규식 확인
3. `max_tokens` 부족 시 응답이 잘림 → 현재 8000으로 설정됨

### 뉴스 수집이 0건일 때

1. Naver API 키 만료 확인 (GitHub Secrets)
2. HEALTH_CONTEXT_WORDS가 너무 좁은지 확인
3. API 호출 제한 (일 25,000회) 초과 여부

### 대시보드가 비어있을 때

1. `data/scans/` 디렉토리에 JSON 파일이 있는지 확인
2. `build_dashboard_v2.py`가 `data/latest.json`으로 폴백하는지 확인
3. `docs/index.html`이 정상 생성되었는지 확인

### 내 블로그 글 목록이 로드 안 될 때

1. `inflow-keyword-analyzer` 레포의 `data/my_posts.json` 존재 여부 확인
2. raw.githubusercontent.com URL 직접 접속해서 JSON 확인
3. 네트워크 타임아웃 (15초) → 빈 목록으로 폴백됨 (기능은 동작하지만 "이미 작성" 판단 불가)
